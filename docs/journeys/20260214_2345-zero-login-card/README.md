## 修复前对比

修复前 (R3 旅程): 所有图层 X 为 -103 ~ -227，原因是拖拽起点落在画板外时
`toArtboard()` 回退到 canvas-world 坐标系，与画板本地坐标系不匹配。

## 修复方案

`useDrawTool.ts` / `useTextTool.ts`:
- `resolveFrame(e)`: 三级查找链确保始终锁定画板 frame
- `bw = 2 * canvas.zoom`: 补偿 CSS border 在 zoom 变换后的像素偏移

## 摩擦点

- **无新增摩擦** (P1 坐标偏移已修复)
- 已知 P2: 文字内容编辑需通过属性面板 textarea (双击画布不进入编辑模式)
