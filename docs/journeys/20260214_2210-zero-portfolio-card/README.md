## 摩擦点

### #1: 数值输入框编辑体验 (观察)

**文件**: `src/composables/useKeyboard.ts:36-44`, `src/components/panels/PropertiesPanel.vue`

**现象**: 在 Playwright headless 环境下，点击数值输入框 (X/Y/W/H) 后通过 keyboard API 输入数字，值未变化。

**分析**:

- `isEditingText()` 排除 `type="number"` 是**有意为之** — 注释明确说明"数值/颜色 input 中工具快捷键仍生效"
- 工具快捷键 (v/f/r/o/t) 在数值输入框中切换工具是合理设计 — 这些字母在数字输入中无效
- 数字键 (0-9) 不受影响，会正常传递给输入框
- 实际问题可能源自 Vue `:value` 单向绑定 + Playwright keyboard API 的交互时序

**严重程度**: 低 — 可能仅影响自动化测试环境，真实用户交互应不受影响。

### #2: macOS 平台 Ctrl+A vs Meta+A

macOS 上 `Ctrl+A` 是行首跳转 (emacs 风格)，`Cmd+A (Meta+A)` 才是全选。这是平台特性，非代码 bug。journey-server 的 AI 操作应使用 `Meta+a` 进行全选。
